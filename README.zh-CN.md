# Release Guardian — 经脱敏的公开重放

一个小型、确定性的包，用于检查发布风险证据在发布决策之前如何被评分、设界和验证。它公开一个虚构重放以及经脱敏的评估表格——而不是私有仓库或 Release Guardian 实现，不是一个实时模型，也不是私有工作流的复现。

## 此处公开的内容

- 四个虚构变更场景和一个确定性评分函数。
- 一个经哈希验证的证据清单，以及被保留的历史候选清单。
- 分别用于资金支持的实机评估、确定性桩评估、注明日期的成本证据和一致性发现的脱敏表格。
- 经脱敏的 Mermaid 架构源码。未包含截图，因为源清单将它们保留为候选项，而非已批准的发布资产。

源代码、原始评估报告、私有文档、内部标识符和工作站元数据均被有意省略。

## 运行确定性重放

需要 Node.js 24 或更新版本。

```bash
npm ci
npm test
npm run verify:evidence
npm run replay -- SYN-AUTH-01
npm run replay -- SYN-SCHEMA-02
```

认证夹具返回风险分数 25（`medium`），且没有阻断项。破坏性模式夹具返回风险分数 90（`critical`），并带有回滚、监控和缺失证据阻断项。这些是固定的合成规则输出。它们既不调用模型，也不复现私有 LangGraph 工作流。

## 必须保持成对的证据

**资金支持的实机证据（已测量，2026-07-11；`evidence/data/evaluation-live.csv`）：全部八个聚合门通过，同时在严格的全试验结果视图中，30/44 个场景被标记。** 该聚合结果并不是声称所有场景都通过。该表覆盖 132 次图运行（44 个场景 × 3 次试验），严格视图还记录了一次轨迹失败。

**确定性桩证据（无 API 密钥或 GPU；`evidence/data/evaluation-stub.csv`）：全部八个聚合门通过，同时在严格的全试验视图中，15/44 个场景被标记。** 桩指标不是资金支持的实机性能；桩代理记账不是 API 支出或实机延迟。

这两种模式是独立的证据类别，不能相互替代。本仓库中的重放是第三个、呈现层工件，并不是任一评估可在此重新运行的证据。

## 成本证据标签

`evidence/data/cost-evidence.csv` 是一个注明日期为 2026-07-08 的迁移前快照。其各行保留原始类别：

- **已测量：** 在相同 token 计数下，路由对比全强成本在 12 次运行中约低 0.25%——不是 25%。
- **已估计：** 一个证据提示剪枝示例将字符数减少了 49.516%，且 token 计数被明确标注为估计。
- **已投影：** 缓存节省取决于所述 10% 缓存输入计费假设。
- **已建模：** ReAct 比较器由观察到的轨迹长度推导而来，而不是作为已执行基线测量得到。

这些注明日期的行都不是当前提供商价格主张，也不是因果性能估计。

## 完整性与边界

`npm run verify:evidence` 会依据 `evidence/manifest.json` 检查每个被复制的资产，验证实机/桩分离和残余披露，并确认历史源清单保持字节级一致。发布记录是追加性的：它不会改写历史清单的候选状态。

本仓库是根据 `evidence/manifest.json` 中限定范围的 2026-07-22 发布记录发布的静态证据快照。它不是一个积极维护的软件项目，也不暗示任何发布节奏、支持或贡献计划。

本包不确立本地复现、仓库默认模型对齐、迁移后完整套件结果、生产就绪性或完整私有源代码谱系。私有原始工件仍被保留；其哈希是来源锚点，而不是分发的文件。未授予开源许可证；不因发布而推定任何许可。
